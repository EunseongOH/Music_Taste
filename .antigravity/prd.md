[PRD] Sortify: 아날로그 감성 LP 인터랙션 취향표 서비스

## 1. 제품 개요 (Product Overview)
* **비전**: "취향을 기록하고, LP로 플레이하다." 단순한 선택을 넘어 아날로그의 물리적 경험을 디지털로 재현하여 나만의 세밀한 음악 취향표를 생성.
* **핵심 컨셉**: Analog Record Shop. 따뜻한 종이 질감과 미니멀한 선화(Line-art)를 결합하여 감성적인 무드 제공.
* **주요 타겟**: SNS 공유를 즐기며, 아날로그 감성과 모던한 디자인을 동시에 선호하는 음악 팬.

---

## 2. 디자인 시스템 (Design System - Option 2 반영)
* **Main Theme**: Warm Cream & Navy (아날로그 종이 질감)
* **Color Palette**:
  * **Background**: `#F5F2ED` (따뜻한 오프 화이트/크림색)
  * **Text/Line-art**: `#2D3436` (깊은 차콜 그레이) 또는 `#1A2A6C` (딥 네이비)
  * **Point Color**: `#E67E22` (레코드 라벨 포인트 오렌지)
* **Typography**: 가독성 좋은 산세리프(San-serif) 계열 + 클래식한 세리프(Serif) 제목 혼용.
* **Vibe**: 따뜻한, 클래식한, 모던한, 손맛이 느껴지는.

## 3. 상세 기능 요구사항 (Functional Requirements)

### [FUNC-01] 온보딩 및 진입
* **01-1 랜딩**: 웜 크림 배경에 세련된 LP 플레이어 애니메이션 노출. "당신의 음악적 온도를 확인해보세요" 문구.
* **01-2 로그인 분기**: 데이터 저장을 위한 로그인 유도 팝업. (Supabase Auth 연동)
  * **이메일 로그인**: 일반 계정 가입 및 로그인.
  * **구글 간편 로그인 (OAuth)**: 구글 계정 연동 로그인 지원.
* **01-3 게스트 모드**: '가입 없이 시작' 버튼 제공. (세션 스토리지 기반 임시 세션 작동)
* **01-4 프로필 헤더 및 상태 관리**:
  * 로그인 성공 시 우측 상단 헤더에 사용자 프로필 이미지(구글 프로필 또는 랜덤 이미지)와 닉네임 노출.
  * 프로필 아이콘 클릭 시 마이페이지 모달이 등장하며, 아래의 **이중 탭(Tab) 인터페이스** 제공:
    * **탭 1: 프로필 수정**: 닉네임/전화번호 수정, 프로필 이미지 변경 기능 및 안전한 **로그아웃** 버튼 제공. 로그아웃 시 로컬/세션스토리지 및 클라우드 데이터를 클리어하고 랜딩 페이지 리다이렉트.
    * **탭 2: 내 아카이브**: 사용자가 그동안 저장했던 역대 최종 취향표 목록(`archives`)을 날짜 및 1위 곡 정보와 함께 리스팅.
      * **상세 보기**: 아카이브 항목 선택 시 모달 내에서 해당 취향표의 전체 랭킹 곡 리스트(순위, 앨범 커버, 곡 정보)를 스크롤 형태로 즉시 확인 가능.
      * **취향표 불러오기 & 공유**: 상세 화면 하단의 버튼 클릭 시 해당 취향표 데이터를 세션에 탑재하고 결과 페이지(`/taste`)로 이동시켜, LP 첫인상 1인칭 드론 애니메이션을 다시 감상하고 인스타그램 스토리용 이미지로 다운로드 및 공유할 수 있도록 지원.
  * 이미 정상 로그인되었거나 게스트 세션이 유효할 경우, 메인화면의 [시작하기] 클릭 시 로그인 모달을 생략하고 즉시 서비스 화면으로 이동.
  * 세션이 로드된 상태에서 랜딩 페이지 진입 시 자동으로 메인 서비스 페이지로 즉시 화면 전환(Redirect) 처리.
* **01-5 클라우드 동기화 및 월드컵 복구 Flow**:
  * 사용자의 진행 상태(선택한 아티스트, 선택한 후보 트랙, 월드컵 진행 현황 등)를 브라우저의 `localStorage`/`sessionStorage`뿐만 아니라 **Supabase 클라우드 데이터베이스(`user_metadata`)에 실시간 이중 동기화**하여 저장.
  * 다른 기기나 새 브라우저로 재진입하거나, 새로고침 등으로 세션이 리셋되더라도 로그인 시점에 클라우드 데이터가 감지되면 **"이어서 진행할까요?"** 팝업 노출.
  * **이어서 진행**: 복제/복구된 대진 정보를 바탕으로 멈췄던 라운드/매치부터 이어서 시작.
  * **새로 시작**: 이전 진행 데이터를 완전히 초기화하고 새로운 아티스트 선택 화면으로 진입.

### [FUNC-02] 취향 탐색 (Spotify API)
* **02-1 아티스트 선택**: 최소 3명 선택. 선택 시 유사 아티스트가 확장되며 리스트에 나타남.
* **02-2 곡 필터링**: 선택한 아티스트의 전체 트랙 중 월드컵 후보곡 선택. 앨범 커버는 부드러운 그림자(Shadow) 효과 적용.
* **02-3 미발매곡 추가**: 사용자가 직접 곡 정보 입력 시 앨범 재킷 사진도 직접 첨부하도록 하며, 미 첨부 시 텍스트 기반의 가상 앨범 아트 자동 생성.

### [FUNC-03] 핵심: LP 월드컵 인터랙션
* **03-1 대진 화면**: 중앙에 두 개의 앨범 커버. 하단에 딥 네이비 선화 스타일의 LP 플레이어 배치.
* **03-2 변환 (Long Press)**: 커버를 꾹 누르면 입체적인 LP 레코드로 변신. (중앙 라벨에 커버 이미지 포함)
* **03-3 드래그 & 드롭**: LP를 하단 플레이어 턴테이블 위로 드래그.
* **03-4 플레이 애니메이션**: 드롭 시 LP 회전 + 톤암 이동 효과 후 승리 곡 기록 및 다음 라운드 전환.

### [FUNC-04] 결과 생성 및 이미지화
* **04-1 취향표 생성**: 1~30위 곡을 트리 혹은 감각적인 리스트 형태로 배치. 배경에 미세한 종이 질감(Grain) 효과 추가.
* **04-2 이미지 저장**: 9:16 비율의 인스타그램 스토리 최적화 이미지 생성 (`html-to-image`).
* **04-3 내 아카이브 저장 및 종료 Flow**:
  * 로그인한 사용자에게는 최종 취향표 화면에 **[내 아카이브에 저장]** 버튼 노출. 클릭 시 취향표 대진 랭킹 데이터가 유저의 Supabase 클라우드 아카이브 목록(`archives`)에 추가 및 보존됨. 이미 저장 완료된 경우 버튼이 비활성화되며 완료 상태를 표기.
  * 헤더 우측에 눈에 띄는 **[종료하기 (X)]** 버튼 배치. 로그인한 사용자가 아직 아카이브에 취향표를 저장하지 않은 상태에서 종료를 시도하면, 저장하지 않고 정말 종료할 것인지 묻는 안전 확인 경고 팝업 제공. 종료 승인 시 진행 중인 모든 세션 데이터 및 클라우드의 임시 진행 백업을 완벽하게 초기화하고, 랜딩 페이지(`/`)로 복귀하여 깔끔하게 새 게임을 시작할 수 있는 상태 제공.

## 4. 기술 스택 (Technical Specs)
* **Frontend**: Next.js (App Router), TypeScript
* **Styling**: Tailwind CSS
* **Animation**: Framer Motion, React Use Gesture
* **Backend / DB**: Supabase (Auth, User Metadata cloud sync)
* **API**: Spotify Web API, Google OAuth API
